# Builder Prompt — TypeScript SDK Phase 8: Testing + Conformance

**Target:** `@aafp/sdk` (pure-TypeScript AAFP SDK)
**Goal:** Prove byte-for-byte wire compatibility with the Rust reference
implementation (`implementations/rust/`) via a comprehensive test and
conformance suite, runnable on Node.js, Deno, and Bun in CI.
**Reference:** `TYPESCRIPT_SDK_DESIGN.md` §13 (Conformance Strategy), §14
Phase 6/8; `implementations/rust/crates/aafp-conformance/`; test vectors in
`test-vectors/mldsa65/`.
**Status:** Frozen wire format — RFC-0002 Rev 6 (Release Candidate).

---

## 1. Objective

Build a testing + conformance layer for the TypeScript SDK that proves:

1. The TS CBOR encoder/decoder produces byte-identical output to
   `aafp-cbor` (RFC 8949 deterministic, integer-keyed maps).
2. The TS frame encoder/decoder produces byte-identical 28-byte headers +
   payloads to `aafp-messaging` (RFC-0002 §3).
3. The TS ML-DSA-65 implementation (`@noble/post-quantum`) cross-verifies
   against the Rust + Go test vectors in `test-vectors/mldsa65/` (A-10).
4. The TS v1 handshake state machine reproduces the canonical handshake
   transcripts (transcript hash, session_id derivation, signature inputs).
5. TS client ↔ Rust server and TS server ↔ Rust client can complete a
   full handshake + RPC exchange over real QUIC.
6. The conformance suite mirrors the Rust `aafp-conformance` crate module
   for module, tagged with RFC normative requirement IDs.
7. Golden traces generated by the Rust `aafp-conformance` crate are
   replayed in TS and asserted byte-equal.

**Definition of done:** CI is green on Node.js, Deno, and Bun; the
conformance suite passes 100% of the Rust golden traces; at least one
cross-language interop test (TS↔Rust) runs in CI.

## 2. Test Framework — Vitest (primary) + runtime smoke tests

Use **Vitest** as the primary unit/integration test framework. Vitest is
chosen over Jest because:

- Native ESM (the SDK ships ESM-first; Jest's CJS-first model fights this).
- Works in Node.js, Deno (via `vitest` CLI), and Bun (`bun run vitest`).
- Built-in support for `describe`/`it`/`expect`, snapshot testing, and
  parameterized `it.each` — needed for the test-vector tables.
- TypeScript support out of the box (no `ts-jest`).

## 3. CBOR Test Vectors (matching Rust `test_vectors.rs`)

The Rust crate `aafp-conformance/src/test_vectors.rs::cbor_vectors()`
produces deterministic CBOR encodings for unsigned integers, negative
integers, byte strings, text strings, arrays, and integer-keyed maps.
Export these to `test/vectors/cbor_vectors.json` via a Rust binary
(`aafp-conformance/src/bin/export_vectors.rs`) that serializes each
`TestVector { name, cbor_bytes, expected_hash }` to JSON.

**TS test — `test/unit/cbor.test.ts`:**

```typescript
import { describe, it, expect } from "vitest";
import { encodeCbor, decodeCbor } from "../../src/cbor.js";
import vectors from "../vectors/cbor_vectors.json" assert { type: "json" };
import { createHash } from "node:crypto";

const toBytes = (hex: string): Uint8Array =>
  Uint8Array.from(hex.match(/.{2}/g)!.map((b) => parseInt(b, 16)));

describe("CBOR golden vectors (Rust aafp-cbor)", () => {
  for (const v of vectors.cbor) {
    it(`vector: ${v.name} [${v.rfc_section}]`, () => {
      const input = JSON.parse(v.semantic_input);
      const encoded = encodeCbor(input);
      // Byte-for-byte equality with Rust output.
      expect(Buffer.from(encoded).toString("hex")).toBe(v.cbor_hex);
      // SHA-256 hash matches the recorded expected_hash.
      const hash = createHash("sha256").update(encoded).digest("hex");
      expect(hash).toBe(v.expected_hash_hex);
      // Round-trip: decode(encoded) === semantic input.
      const decoded = decodeCbor(encoded);
      expect(decoded).toEqual(input);
    });
  }

  it("integer-keyed map uses canonical ordering (RFC 8949 §3.1)", () => {
    // Keys MUST be sorted in canonical byte-lex order, NOT numeric order.
    // Key 100 (0x18 0x64) sorts BEFORE key 23 (0x17) is FALSE —
    // 0x17 < 0x18, so 23 encodes first. Verify Rust matches.
    const map = new Map<number, unknown>([[100, "a"], [23, "b"], [1, "c"]]);
    const encoded = encodeCbor({ type: "intmap", entries: map });
    // Expected: 0xA3 0x01 0x61'c' 0x17 0x61'b' 0x1864 0x61'a'
    expect(Buffer.from(encoded).toString("hex"))
      .toBe("a301616317616218646161");
  });

  it("deterministic encoding: same value → same bytes", () => {
    const v = { type: "intmap", entries: new Map([[1, "x"], [2, "y"]]) };
    expect(Buffer.from(encodeCbor(v)).toString("hex"))
      .toBe(Buffer.from(encodeCbor(v)).toString("hex"));
  });
});
```

**Coverage targets:** every `TestVector` in `cbor_vectors()` (unsigned,
negative, byte string, text, array, nested map, empty map, large u64
requiring 8-byte additional info). The `it.each` loop above iterates the
exported JSON so adding a Rust vector automatically adds a TS test.

## 4. ML-DSA-65 Cross-Language Test Vectors (A-10)

The Rust + Go implementations produce identical ML-DSA-65 keys from the
same 32-byte seed and identical deterministic signatures. The TS SDK
must join this cross-verification matrix using `@noble/post-quantum`.

**Vector sources (in `test-vectors/mldsa65/`):**
- `vectors.json` — 19 Rust-generated vectors (seed → keypair → signature).
- `go_vectors.json` — 15 Go-generated vectors.
- `diff_traces.json` / `go_diff_traces.json` — 100 diff traces each.

Each vector: `{ id, seed, message_hex, context_hex, public_key_hex,
secret_key_hex, signature_hex, expected_verify, description }`.

**TS test — `test/conformance/mldsa_cross.test.ts`:**

```typescript
import { describe, it, expect } from "vitest";
import { ml_dsa65 } from "@noble/post-quantum";
import { readFile } from "node:fs/promises";

const hex = (s: string): Uint8Array =>
  Uint8Array.from(s.match(/.{2}/g)!.map((b) => parseInt(b, 16)));
const toHex = (b: Uint8Array): string =>
  Array.from(b).map((x) => x.toString(16).padStart(2, "0")).join("");

const load = async (p: string) =>
  JSON.parse(await readFile(p, "utf8")) as any[];

const VECTORS = [
  "test-vectors/mldsa65/vectors.json",       // Rust
  "test-vectors/mldsa65/go_vectors.json",    // Go
];

describe("ML-DSA-65 cross-language verification (A-10)", () => {
  // 1. Rust + Go signatures verify in TS (@noble/post-quantum).
  for (const path of VECTORS) {
    const label = path.includes("go_") ? "Go" : "Rust";
    const vectors = await load(path);

    it.each(vectors)(
      `${label} vector $id verifies in TS`,
      async (v) => {
        const pk = hex(v.public_key_hex);
        const sig = hex(v.signature_hex);
        const msg = hex(v.message_hex);
        const valid = ml_dsa65.verify(sig, msg, pk);
        expect(valid).toBe(v.expected_verify);
      },
    );
  }

  // 2. Same seed → same public key (keygen determinism).
  it("keygen: seed → public key matches Rust/Go byte-for-byte", async () => {
    const rust = await load("test-vectors/mldsa65/vectors.json");
    const seen = new Set<string>();
    for (const v of rust) {
      if (!v.expected_verify || !seen.add(v.seed)) continue;
      const seed = hex(v.seed);
      const { publicKey } = ml_dsa65.keypair_from_seed(seed);
      expect(toHex(publicKey)).toBe(v.public_key_hex);
    }
  });

  // 3. Deterministic signing: same secret key + message → same signature.
  it("deterministic sign: matches Rust signature bytes", async () => {
    const rust = await load("test-vectors/mldsa65/vectors.json");
    for (const v of rust) {
      if (!v.expected_verify) continue;
      const sk = hex(v.secret_key_hex);
      const msg = hex(v.message_hex);
      const sig = ml_dsa65.sign(msg, sk); // deterministic mode
      expect(toHex(sig)).toBe(v.signature_hex);
    }
  });

  // 4. Negative: tampered signature / message / pubkey must fail.
  it("negative: flipped signature bit fails verify", async () => {
    const [v] = await load("test-vectors/mldsa65/vectors.json");
    const pk = hex(v.public_key_hex);
    const sig = hex(v.signature_hex);
    const msg = hex(v.message_hex);
    sig[0] ^= 0x01;
    expect(ml_dsa65.verify(sig, msg, pk)).toBe(false);
  });
});
```

**Acceptance criteria:** 19/19 Rust vectors verify in TS, 15/15 Go
vectors verify in TS, keygen matches for all unique seeds, deterministic
signatures match byte-for-byte. This closes the A-10 matrix:
Rust↔Go↔TypeScript.

## 5. Handshake Test Vectors

The Rust `aafp-conformance/src/handshake_vectors.rs` produces a complete
`HandshakeVector` with: fixed nonces, TLS binding, serialized public
keys, transcript checkpoints (after ClientHello / ServerHello /
ClientFinished), CBOR encodings of each message, signature inputs,
signatures, and the derived `session_id`.

Export to `test/vectors/handshake_vectors.json` and replay in TS.

**TS test — `test/unit/handshake.test.ts`:**

```typescript
import { describe, it, expect } from "vitest";
import { encodeCbor } from "../../src/cbor.js";
import { TranscriptHash, deriveSessionId } from "../../src/handshake.js";
import { ml_dsa65 } from "@noble/post-quantum";
import { createHash } from "node:crypto";
import hv from "../vectors/handshake_vectors.json" assert { type: "json" };

const hex = (s: string): Uint8Array =>
  Uint8Array.from(s.match(/.{2}/g)!.map((b) => parseInt(b, 16)));
const sha256 = (b: Uint8Array) => createHash("sha256").update(b).digest();

describe("v1 handshake transcript (RFC-0002 §5)", () => {
  it("ClientHello CBOR matches Rust byte-for-byte", () => {
    const ch = buildClientHello(hv); // from fixed inputs
    expect(Buffer.from(encodeCbor(ch.toCbor())).toString("hex"))
      .toBe(hv.client_hello_cbor_hex);
  });

  it("transcript hash after ClientHello matches", () => {
    const t = TranscriptHash.fromTlsBinding(hex(hv.tls_binding_hex));
    const h = t.fold(hex(hv.client_hello_cbor_hex));
    expect(Buffer.from(h).toString("hex"))
      .toBe(hv.transcript_after_client_hello_hex);
  });

  it("session_id derivation matches Rust", () => {
    const h = hex(hv.transcript_after_client_finished_hex);
    const sid = deriveSessionId(h);
    expect(Buffer.from(sid).toString("hex")).toBe(hv.session_id_hex);
  });

  it("ClientHello signature verifies with recorded public key", () => {
    const pk = hex(hv.client_public_key_hex);
    const sigInput = hex(hv.client_hello_sig_input_hex);
    const sig = hex(hv.client_hello_signature_hex);
    expect(ml_dsa65.verify(sig, sigInput, pk)).toBe(true);
  });

  it("ServerHello + ClientFinished signatures verify", () => {
    const spk = hex(hv.server_public_key_hex);
    expect(ml_dsa65.verify(
      hex(hv.server_hello_signature_hex),
      hex(hv.server_hello_sig_input_hex), spk)).toBe(true);
    const cpk = hex(hv.client_public_key_hex);
    expect(ml_dsa65.verify(
      hex(hv.client_finished_signature_hex),
      hex(hv.client_finished_sig_input_hex), cpk)).toBe(true);
  });
});
```

Additionally, port the Rust `handshake_state_machine.rs` tests: invalid
transitions, duplicate ClientHello, unexpected frame rejection, timeout
behavior. Each TS test is tagged with the RFC-0002 §5.10 sub-state it
covers.

## 6. Frame Encoding/Decoding Tests

Mirror `aafp-conformance/src/rfc0002.rs` (frame header tests R2-001
through R2-0xx). The 28-byte header layout:

| Offset | Field | Size |
|--------|-------|------|
| 0 | protocol version | 1 |
| 1 | frame type | 1 |
| 2 | flags | 1 |
| 3 | reserved (MUST be 0) | 1 |
| 4 | stream id | 8 (BE) |
| 12 | payload length | 8 (BE) |
| 20 | extension length | 8 (BE) |

**TS test — `test/unit/frame.test.ts`:**

```typescript
import { describe, it, expect } from "vitest";
import { encodeFrame, decodeFrame, Frame, FrameType, AAFP_VERSION }
  from "../../src/frame.js";

describe("RFC-0002 §3 frame header", () => {
  it("R2-001: header is 28 bytes", () => {
    const f = Frame.data(1n, new Uint8Array([0xab]));
    expect(encodeFrame(f)[0..28]).toBeDefined();
    expect(encodeFrame(f).length).toBe(28 + 1);
  });

  it("R2-002: version at offset 0", () => {
    const bytes = encodeFrame(Frame.data(0n, new Uint8Array(0)));
    expect(bytes[0]).toBe(AAFP_VERSION);
  });

  it("R2-003: frame type at offset 1", () => {
    const bytes = encodeFrame(Frame.data(0n, new Uint8Array(0)));
    expect(bytes[1]).toBe(FrameType.Data);
  });

  it("R2-006: stream id is 8-byte big-endian at offset 4", () => {
    const sid = 0x123456789abcdef0n;
    const bytes = encodeFrame(Frame.data(sid, new Uint8Array(0)));
    const view = new DataView(bytes.buffer, 4, 8);
    expect(view.getBigUint64(0)).toBe(sid);
  });

  it("R2-007: payload length 8-byte BE at offset 12", () => {
    const payload = new Uint8Array(100);
    const bytes = encodeFrame(Frame.data(0n, payload));
    const view = new DataView(bytes.buffer, 12, 8);
    expect(view.getBigUint64(0)).toBe(100n);
  });

  it("R2-005: reserved byte MUST be zero", () => {
    const bytes = encodeFrame(Frame.data(0n, new Uint8Array(0)));
    expect(bytes[3]).toBe(0);
  });

  it("round-trip: decode(encode(f)) deep-equals f", () => {
    const f = Frame.data(42n, new Uint8Array([1, 2, 3, 4]));
    expect(decodeFrame(encodeFrame(f))).toEqual(f);
  });

  it("rejects payload > MAX_PAYLOAD_SIZE", () => {
    const big = new Uint8Array(MAX_PAYLOAD_SIZE + 1);
    expect(() => encodeFrame(Frame.data(0n, big))).toThrow(/payload too large/);
  });

  it("rejects non-zero reserved byte on decode", () => {
    const bytes = encodeFrame(Frame.data(0n, new Uint8Array(0)));
    bytes[3] = 0xff;
    expect(() => decodeFrame(bytes)).toThrow(/reserved/);
  });
});
```

## 7. Integration Tests (TS ↔ Rust, TS ↔ TS)

These require a real QUIC transport. On Node 25+ use `node:quic`
(behind `--experimental-quic`). The Rust side runs as a subprocess
spawned by the test.

**`test/integration/ts_client_rust_server.test.ts`:**

```typescript
import { describe, it, expect, beforeAll, afterAll } from "vitest";
import { spawn, ChildProcess } from "node:child_process";
import { Agent, Request } from "../../src/index.js";

let rustServer: ChildProcess;
let serverAddr: string;

beforeAll(async () => {
  // Start the Rust echo agent from the prebuilt binary.
  rustServer = spawn("./bin/aafp-echo-agent", ["--bind", "127.0.0.1:0"]);
  const ready = await readLine(rustServer.stdout); // "listening 127.0.0.1:PORT"
  serverAddr = ready.split(" ")[1];
}, 60_000);

afterAll(() => rustServer?.kill("SIGTERM"));

describe("TS client ↔ Rust server interop", () => {
  it("completes v1 handshake and echoes a message", async () => {
    const client = await Agent.connect().connect();
    const res = await client.discover("echo")
      .callAt(serverAddr, Request.text("hello from TS"));
    expect(res.body).toBe("hello from TS");
    await client.stop();
  });

  it("v2 Params round-trip across languages", async () => {
    const client = await Agent.connect().connect();
    const req = Request.withParams(
      Params.create().putStr(1, "sum").putU64(2, 40n).putU64(3, 2n));
    const res = await client.discover("sum").callAt(serverAddr, req);
    expect(res.params.getU64(1)).toBe(42n);
  });
});
```

**`test/integration/ts_server_rust_client.test.ts`** mirrors this: the

**`test/integration/streaming_interop.test.ts`** verifies a TS streaming

## 8. Conformance Test Suite (mirroring `aafp-conformance`)

Create one TS test module per Rust conformance module, preserving the
RFC requirement IDs (R2-001, R3-002, R5-003, etc.) as test names so a
side-by-side audit is trivial.

| Rust module | TS module | RFC coverage |
|-------------|-----------|--------------|
| `rfc0002.rs` | `conformance/rfc0002.test.ts` | framing, CBOR, handshake transcript |
| `rfc0003.rs` | `conformance/rfc0003.test.ts` | identity, ML-DSA-65, AgentId derivation |
| `rfc0005.rs` | `conformance/rfc0005.test.ts` | error model, HandlerError categories |
| `close_conformance.rs` | `conformance/close.test.ts` | CLOSE frame state machine (§6.6) |
| `replay_conformance.rs` | `conformance/replay.test.ts` | ReplayCache (§6.7) |
| `version_negotiation.rs` | `conformance/version.test.ts` | protocol version negotiation |
| `pipeline_order.rs` | `conformance/pipeline.test.ts` | RPC pipeline ordering rules |
| `negative.rs` | `conformance/negative.test.ts` | malformed input rejection |

Each TS test asserts the same normative property as its Rust counterpart.
Example for RFC-0005 error categories:

```typescript
// conformance/rfc0005.test.ts
import { describe, it, expect } from "vitest";
import { HandlerError } from "../../src/errors.js";

describe("RFC-0005 error model", () => {
  it("R5-001: 8 error categories defined", () => {
    expect(Object.keys(HandlerError.Category).length).toBe(8);
  });

  it("R5-003: category encodes as u8 on wire", () => {
    const e = new HandlerError(HandlerError.Category.NotFound, "no such cap");
    const encoded = e.toCbor();
    expect(encoded[0]).toBe(0x18); // u8 follows
    expect(encoded[1]).toBe(HandlerError.Category.NotFound);
  });

  it("R5-004: unknown category code rejected on decode", () => {
    expect(() => HandlerError.fromCbor(Uint8Array.of(0x18, 0xff)))
      .toThrow(/unknown error category/);
  });
});
```

## 9. Golden Trace Verification

The Rust `aafp-conformance` crate emits golden trace files: full
byte-level transcripts of frame sequences, CBOR encodings, and handshake
messages. These are the single source of truth for wire compatibility.

**Generation (Rust side):** `aafp-conformance/src/bin/export_golden.rs`
writes `test/vectors/golden/*.trace` files, each containing:

```json
{
  "name": "handshake_full_v1",
  "rfc_section": "RFC-0002 §5",
  "frames": [
    { "type": "HANDSHAKE", "hex": "...", "direction": "C->S" },
    { "type": "HANDSHAKE", "hex": "...", "direction": "S->C" },
    { "type": "HANDSHAKE", "hex": "...", "direction": "C->S" }
  ],
  "transcript_hash_final": "...",
  "session_id": "..."
}
```

**TS replay — `test/conformance/golden_trace.test.ts`:**

```typescript
import { describe, it, expect } from "vitest";
import { decodeFrame } from "../../src/frame.js";
import { readdirSync, readFileSync } from "node:fs";
import { join } from "node:path";

const dir = "test/vectors/golden";
const traces = readdirSync(dir).filter((f) => f.endsWith(".trace"));

describe("golden trace replay (Rust source of truth)", () => {
  it.each(traces)("trace %s decodes cleanly", (file) => {
    const trace = JSON.parse(readFileSync(join(dir, file), "utf8"));
    for (const frame of trace.frames) {
      const bytes = Uint8Array.from(
        Buffer.from(frame.hex, "hex"));
      // MUST decode without error and re-encode to identical bytes.
      const decoded = decodeFrame(bytes);
      const reencoded = encodeFrame(decoded);
      expect(Buffer.from(reencoded).toString("hex")).toBe(frame.hex);
    }
  });

  it("handshake trace reproduces recorded session_id", () => {
    const trace = JSON.parse(readFileSync(
      join(dir, "handshake_full_v1.trace"), "utf8"));
    const transcript = TranscriptHash.fromTlsBinding(/* from trace */);
    for (const f of trace.frames) {
      transcript.fold(Buffer.from(f.hex, "hex"));
    }
    expect(transcript.sessionIdHex()).toBe(trace.session_id);
  });
});
```

**Rule:** if a TS test diverges from a golden trace, the TS code is
wrong (the Rust reference is the source of truth). Never edit a golden
trace to make a TS test pass; file a bug against the TS implementation.

## 10. CI Pipeline (GitHub Actions)

A single workflow matrix-tests the SDK on **Node.js 22/24/25**, **Deno
2.x**, and **Bun 1.x**. Cross-language interop jobs run only on Linux
(where the Rust binaries are prebuilt) to avoid macOS/Windows Rust
spawn complexity.

**`.github/workflows/ts-sdk-ci.yml`:**

```yaml
name: TS SDK CI

on:
  push:
    paths: ["implementations/typescript/**", "test-vectors/**"]
  pull_request:

jobs:
  unit:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        runtime: [node-22, node-24, node-25, deno, bun]
    steps:
      - uses: actions/checkout@v4
      - name: Setup Node 25 (for QUIC)
        if: matrix.runtime == 'node-25'
        uses: actions/setup-node@v4
        with: { node-version: 25 }
      - name: Setup Deno
        if: matrix.runtime == 'deno'
        uses: denoland/setup-deno@v2
        with: { deno-version: v2.x }
      - name: Setup Bun
        if: matrix.runtime == 'bun'
        uses: oven-sh/setup-bun@v2
      - run: npm ci
        if: startsWith(matrix.runtime, 'node')
      - name: Unit + conformance (Node)
        if: startsWith(matrix.runtime, 'node')
        run: npx vitest run --config vitest.config.ts
      - name: Unit (Deno)
        if: matrix.runtime == 'deno'
        run: deno run --allow-read --allow-env npm:vitest run
      - name: Unit (Bun)
        if: matrix.runtime == 'bun'
        run: bun run vitest run

  interop:
    runs-on: ubuntu-latest
    needs: unit
    steps:
      - uses: actions/checkout@v4
      - uses: actions-rust-lang/setup-rust-toolchain@v1
      - uses: actions/setup-node@v4
        with: { node-version: 25 }
      - name: Build Rust interop binaries
        run: cargo build --release -p aafp-cli -p aafp-echo-agent
      - name: Export Rust test vectors + golden traces
        run: cargo run --release -p aafp-conformance --bin export_golden
      - name: TS ↔ Rust interop
        run: npx vitest run --config vitest.config.ts test/integration
        env:
          AAFP_RUST_BIN: ./target/release
          NODE_OPTIONS: "--experimental-quic"

  conformance-golden:
    runs-on: ubuntu-latest
    needs: unit
    steps:
      - uses: actions/checkout@v4
      - uses: actions-rust-lang/setup-rust-toolchain@v1
      - uses: actions/setup-node@v4
        with: { node-version: 25 }
      - run: cargo run --release -p aafp-conformance --bin export_vectors
      - run: npx vitest run test/conformance
```

**Smoke jobs** (`test/smoke/deno.smoke.ts`, `bun.smoke.ts`) do a minimal
`Agent.connect()` + `discover().call()` against a TS echo server to
confirm the SDK loads and runs in each runtime without full test
coverage. These gate releases.
