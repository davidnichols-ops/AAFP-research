## Research Insights

- **WhatsApp:** 2M connections per server (Erlang). AAFP should target at least 10K connections per agent for v1.
- **Discord:** 5M concurrent users, 15K sessions per relay. AAFP relay default max_connections=50 is very conservative — increase to 1000+ for production.
- **Resilience patterns:** Circuit breaker (5 failures → open, 30s → half-open), bulkhead (limit concurrent requests per peer to 10), retry with exponential backoff + jitter (max 3 retries). Implement directly in aafp-sdk.
