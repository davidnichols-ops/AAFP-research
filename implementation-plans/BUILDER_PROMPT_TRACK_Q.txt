### Q4: Resource exhaustion testing

Create `crates/aafp-tests/tests/resource_exhaustion.rs` with 6 DoS tests:

1. **Connection flood:** Open 1000 connections in 1 second. Measure CPU/memory/FDs. Add `max_connections` config (default 100).
2. **Stream exhaustion:** Open 1000 bi-streams on one connection. Verify quinn enforces `max_streams_bidi`.
3. **Large frame attack:** Send frame with payload_len = 1GB (header only). Verify rejected immediately. Verify 1MB frame works (at limit).
4. **Slow loris:** Open connection, send 1 byte/second. Verify closed after `max_idle_timeout` (30s).
5. **Memory exhaustion:** Send many large messages. Measure memory. Add backpressure if needed.
6. **CPU exhaustion:** Send many handshake requests (ML-DSA-65 verify is expensive). Add rate limiting (10/second per IP).

Write results to `test-results/security/resource-exhaustion.json`.
